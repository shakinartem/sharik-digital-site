# Telegram-бот @sharik_digital_bot: MVP-сценарий

## Главное ограничение

Telegram-бот автоматически получает `id`, `username`, `first_name`, `last_name`, `language_code`. Номер телефона автоматически не отдаётся. Номер можно получить только добровольно через кнопку `request_contact` или ручной ввод.

В MVP номер не просим. Заявка создаётся по Telegram-профилю и ответам диагностики.

## Воронка

1. Пользователь с сайта нажимает “Забрать чек-лист в Telegram”.
2. Бот сохраняет Telegram-профиль и источник.
3. Бот задаёт мягкие вопросы:
   - название клиники;
   - город;
   - роль.
4. Бот выдаёт PDF-чек-лист.
5. Бот предлагает мини-диагностику.
6. После диагностики предлагает “Получить разбор клиники”.
7. Заявка уходит в отдельный внутренний чат команды.

## Формат заявки в чат команды

**Новая заявка с сайта / Telegram-бота**

**Telegram ID:** `{telegram_id}`  
**Username:** `@{username}`  
**Имя:** `{first_name} {last_name}`  
**Клиника:** `{clinic_name}`  
**Город:** `{city}`  
**Роль:** `{role}`  
**Тип клиники:** `{clinic_type}`  
**Что уже есть:** `{existing_tools}`  
**Главная проблема:** `{main_problem}`  
**Куда приходят заявки:** `{lead_channels}`  
**Скорость ответа:** `{response_speed}`  
**Приоритет:** `{priority}`  
**Что хочет разобрать:** `{audit_focus}`  
**Можно написать в Telegram:** `{telegram_contact_allowed}`  
**Комментарий:** `{comment}`
