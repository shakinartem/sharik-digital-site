# AGENTS.md — ШАРиК-digital

Этот файл задаёт правила для Codex на всём проекте. Читай его перед каждой задачей.

---

## Проект

**ШАРиК-digital** — агентство digital-маркетинга для стоматологических клиник.

**Аудитория сайта:** владельцы и управляющие стоматологий (35–55 лет), принимают решения о маркетинговом бюджете. Ценят конкретику: цифры, кейсы, понятные условия работы. Не любят размытые обещания.

**Главная цель сайта:** конвертировать посетителя в заявку на консультацию.

**Главный CTA:** «Оставить заявку на консультацию» — присутствует в hero, после блока услуг и в футере. Кнопка всегда одна и та же по тексту.

**Ключевые смысловые блоки:** hero, результаты/выводы, разбор, кейсы, процесс, услуги, форма заявки, директор/основатель, футер.

**Визуальные референсы:** светлый фон, много воздуха, тонкие разделители, крупная типографика, акцентные бордово-синие элементы, минимум декоративного шума, аккуратные иконки и схемы вместо тяжёлых карточек.

---

## Стек

- **Next.js** (App Router) + TypeScript
- Стилизация: Tailwind CSS, если явно не попросят иначе
- Роутинг: файловый роутинг Next.js
- Формы: нативный `fetch` к бэкенду, без лишних библиотек
- Деплой: Vercel по умолчанию, Netlify только если явно требуется

## Фирменный стиль

> Заполни значения ниже своими данными перед первым запуском Codex.

```css
--color-primary:     #061C41;   /* основной фирменный цвет */
--color-accent:      #9B002F;   /* акцент / CTA-кнопки */
--color-bg:          #E7E7E7;   /* фон страницы */
--color-text:        #061F3F;   /* основной текст */
--color-text-muted:  #2A132D;   /* второстепенный текст */

--font-display: 'Montserrat Rounded', sans-serif;  /* заголовки */
--font-body:    'Manrope', sans-serif;  /* основной текст */
```

**Правила по стилю:**
- Всегда используй CSS-переменные выше — никаких хардкодных hex-значений в компонентах
- Максимум 2 шрифта: display для заголовков, body для всего остального
- Один акцентный цвет — только для CTA-кнопок и ключевых выделений
- Никаких карточек с тенями без явной просьбы — предпочитай секции, разделители, колонки

## Дизайн-правила (обязательно перед вёрсткой)

Перед написанием кода любого нового экрана или компонента сформулируй три вещи:

1. **Визуальный тезис** — одно предложение: настроение, материал, энергия страницы
2. **Контент-план** — hero / доказательство / детали / финальный CTA
3. **Анимации** — 2–3 конкретных момента (не «везде», а точечно)

**Жёсткие правила вёрстки:**
- Hero — полноэкранный, бренд и оффер читаются за 3 секунды
- Логотип/название — самый заметный текстовый элемент первого экрана
- Фон не должен быть однотонным: градиент, текстура или сильное фото
- Типографика выразительная — запрещены Inter, Roboto, Arial как display-шрифт
- Cardless по умолчанию: используй whitespace, выравнивание и контраст вместо карточек
- На мобиле (375px) всё должно читаться без горизонтального скролла

## Структура сайта

```
/                  — лендинг (главная точка конверсии)
  #hero            — оффер + CTA
  #why-us          — почему стоматологиям нужен специализированный маркетинг
  #services        — услуги (SEO, контекст, соцсети, сайты для клиник)
  #cases           — 2–3 кейса с цифрами
  #how-we-work     — процесс работы (3–4 шага)
  #faq             — частые вопросы владельцев клиник
  #contact         — форма заявки на консультацию
```

## Тон и копирайтинг

- Язык: русский, деловой но не официальный — как разговор партнёра с партнёром
- Конкретика вместо абстракций: «+34 новых пациента за 2 месяца», не «увеличим поток клиентов»
- Никаких клише: «команда профессионалов», «индивидуальный подход», «под ключ» — запрещены
- CTA всегда активный глагол: «Оставить заявку», «Получить аудит», «Обсудить проект»
- Заголовки секций — короткие, не больше 6 слов

---

## Компоненты: соглашения

- Один файл = один компонент, папка `components/`
- Секции страницы — в `app/` или в локальных компонентах по смыслу страницы
- Имена файлов: PascalCase (`HeroSection.tsx`, `ContactForm.tsx`)
- Каждый компонент экспортируется как default
- Пропсы описываются через TypeScript interface прямо в файле компонента
- Никаких inline-стилей кроме CSS-переменных через `style={{ '--var': value }}`

---

## Форма заявки

Поля: имя, телефон, название клиники (опционально), комментарий (опционально).
Отправка: `fetch` POST на `/api/contact` (или заглушка до подключения бэкенда).
После отправки: показать inline-сообщение об успехе, не перезагружать страницу.
Валидация: только имя и телефон обязательны, телефон — российский формат.
