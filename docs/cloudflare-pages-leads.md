# Заявки с сайта

Форма сайта отправляет данные на Cloudflare Pages Function:

```
POST /api/lead
```

## Переменные Cloudflare Pages

Для работы функции нужны переменные окружения:

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен Telegram-бота |
| `ADMIN_CHAT_ID` | ID чата для уведомлений |

Если переменные не заданы, функция вернёт `500` с ошибкой.

## Как это работает

1. Пользователь заполняет форму пред-аудита на сайте.
2. Форма отправляет `POST /api/lead` с JSON-телом.
3. Cloudflare Pages Function отправляет сообщение в `ADMIN_CHAT_ID` через Telegram Bot API.
4. Пользователь видит подтверждение отправки и кнопку перехода в Telegram-бота.
5. Если ошибка — пользователь видит кнопку «Попробовать снова» и запасной вариант перехода в Telegram.

## Важно

Данные из `localStorage` браузера не могут автоматически попасть в Telegram-бота. Поэтому сайт сначала отправляет заявку серверно, а потом предлагает продолжить диагностику в боте.

## Формат сообщения в Telegram

```
🦷 Новая заявка с сайта ШАРиК-digital

Источник: {source}
Страница: {page}

Имя: {name}
Клиника: {clinic}
Город: {city}
Удобный канал: {contact}

Что хочет улучшить:
{improvement}

Комментарий:
{comment}

Дата: {isoDate}
User-Agent: {userAgent}
```

## Honeypot-защита

В форме есть скрытое поле `website`. Если бот его заполнит, функция вернёт `{ "ok": true }`, но не отправит сообщение в Telegram.
