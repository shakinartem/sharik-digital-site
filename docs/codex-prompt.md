# Codex prompt для запуска разработки

Ты работаешь в репозитории `sharik-digital-site`.

Цель: довести MVP-лендинг ШАРиК-digital до готового деплоя.

## Что уже есть

- Next.js + TypeScript + Tailwind CSS.
- Главная страница с блоками:
  - Hero;
  - проблема потери пациентов;
  - цепочка потерь;
  - digital-система;
  - лид-магнит;
  - диагностика;
  - 9 кейсов с модальными окнами;
  - доверие;
  - основатель;
  - процесс;
  - услуги;
  - комплексы;
  - FAQ;
  - финальный CTA;
  - футер.
- Ассеты в `public/brand`, `public/refs`, `public/cases`.
- Данные кейсов в `data/cases.ts`.
- Основные настройки сайта в `data/site.ts`.

## Приоритеты

1. Проверить сборку `npm run build`.
2. Исправить TypeScript/Tailwind ошибки, если есть.
3. Улучшить адаптивность мобильной версии.
4. Проверить, что все CTA ведут на `https://t.me/sharik_digital_bot`.
5. Проверить модальные окна кейсов.
6. Добавить плавные hover/focus состояния.
7. Подготовить деплой на Vercel или Cloudflare Pages.

## Важно

- Не удалять кейсы и названия клиник.
- Услуги и комплексы должны быть ближе к концу страницы.
- Лид-магнит должен быть высоко на странице.
- Главная идея сайта: сначала ценность и диагностика, потом услуги.
- Номер телефона на сайте можно показывать, но в боте в MVP номер не требовать.
