## Проект

**ШАРиК-digital** — digital-система для стоматологических клиник: чек-лист, Telegram-бот, мини-диагностика, разбор клиники и внедрение решений.

**Аудитория сайта:** владельцы и управляющие стоматологий 35–55 лет. Они принимают решения о маркетинговом бюджете, ценят конкретику, цифры, кейсы и понятные шаги. Не любят размытые обещания и длинные формы на первом касании.

**Главный смысл сайта:** находим, где стоматология теряет пациентов, и собираем digital-систему, которая доводит человека от первого касания до заявки и записи.

**Основная воронка:** сайт → Telegram-бот → чек-лист → мини-диагностика → разбор клиники.

**Главный CTA:** «Забрать чек-лист в Telegram».

**Второй CTA:** «Понять, где теряются пациенты».

Телефон не обязателен. Форма с телефоном не является главным сценарием MVP. Основные CTA ведут в Telegram-бота.

## Фирменный стиль

```css
--color-primary:     #061C41;
--color-accent:      #9B002F;
--color-bg:          #E7E7E7;
--color-text:        #061F3F;
--color-text-muted:  #2A132D;

--font-display: 'Montserrat Rounded', sans-serif;
--font-body:    'Manrope', sans-serif;
```

**Правила по стилю:**
- Всегда используй CSS-переменные бренда, не хардкодь hex в компонентах без причины
- Максимум 2 шрифта: display для заголовков, body для остального
- Один акцентный цвет для CTA и ключевых выделений
- Светлый фон, тёмно-синий текст, бордовые dotted/line элементы
- Карточки кейсов и модальные окна сохраняем
- Не превращай сайт в обычный лендинг SMM/сайтов/рекламы

## Дизайн-правила

Перед новым экраном или крупным компонентом сформулируй:

1. **Визуальный тезис** — настроение, материал, энергия
2. **Контент-план** — hero / ценность / лид-магнит / диагностика / доказательство / CTA
3. **Анимации** — 2–3 точечных момента, без лишнего шума

**Жёсткие правила:**
- Hero объясняет “где теряются пациенты” за 3 секунды
- Главный путь — Telegram-бот, а не обязательная форма
- Лид-магнит и диагностика идут высоко
- Услуги и решения ближе к концу
- На 375px не должно быть горизонтального скролла
- Маскот не перекрывает текст и кнопки

```
  #services        — услуги
  #solutions       — решения / комплексы
  #contact         — финальный Telegram-first CTA
```

## Копирайтинг

- Язык: русский, деловой, партнёрский
- Конкретика вместо абстракций
- Позиционирование: “где теряются пациенты”, “маршрут пациента”, “digital-система”
- Не использовать клише: «команда профессионалов», «индивидуальный подход», «под ключ»
- CTA должны вести к действию в Telegram: чек-лист, диагностика, разбор, вопрос

## Компоненты

- Компоненты — в `components/`
- Секции страницы могут жить в `app/page.tsx`, пока страница компактна
- Имена файлов: PascalCase
- Пропсы описываются через TypeScript interface или inline type
- Inline-стили только для CSS-переменных, если без них нельзя

## Проверка

Перед финальным ответом по UI-задаче:

- `npm run build`
- `npm run lint`, если доступен
- Playwright-проверка desktop/mobile
- Проверить CTA deep links
- Проверить, что мобильное меню работает
- Проверить, что кейсы открываются и закрываются, Escape закрывает модалку
- Проверить отсутствие горизонтального скролла на мобильной ширине
- Основные брендовые изображения: `public/brand/mascot-balloon.png` и `public/brand/founder-artem.png`

## Чего НЕ делать

- Не возвращай главный CTA «Оставить заявку на консультацию»
- Не делай телефон обязательным
- Не поднимай услуги выше чек-листа и диагностики
- Не добавляй зависимости без явной просьбы
- Не меняй фирменные цвета без явной просьбы
- Не деплой без подтверждения
